§ 核心叙事

统一 packet 链路

这是整个项目最重要的设计事实。HTTP 与 WebSocket 不再是两套割裂入口，而是汇入同一条命令分发模型；Gateway 只做协议入口与转发，业务解析下沉到各 service 的 packet dispatcher。

main pipeline
# HTTP 与 WebSocket 殊途同归
HTTP / WebSocket
  → MessageParser          # 解析为 UnifiedMessage
  → MessageProcessor       # 投递线程池，按 cmd_id 找 handler
  → CommandHandlerRegistry # 鉴权 + 注入可信身份到 packet header
  → RuntimeRegistry        # 选择 local service 或 remote adapter
  → XxxPacketDispatcher    # service 内解析 payload、执行业务规则
  → PostgreSQL / Redis / Push
① 协议入口统一

HTTP catch-all 与 WS envelope 都经 MessageParser 转成 UnifiedMessage，路由来自配置而非硬编码 controller。

② 鉴权边界清晰

Gateway 校验 token 并把可信 uid/device/platform 写入 packet header；service 不信任 body 中的发送者。

③ 部署模式可切

同一套 handler 通过 RuntimeRegistry 选择 local dispatcher 或 gRPC ForwardPacket，从 all-in-one 平滑演进到分布式。
