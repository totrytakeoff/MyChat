# MyChat 项目总结分析与面试准备总规划

日期：2026-06-23

## 1. 阶段目标

MyChat 当前进入“稳定 MVP + 架构总结 + 面试准备”阶段。后续重点不是继续堆功能，而是把已经完成的分布式 IM 后端梳理成一套可以运行、可以演示、可以讲清楚、可以经受追问的工程作品。

本阶段目标：

- 测试验证：围绕单机、Web 双账号、多进程 remote-all、多机器部署做验证。
- 面试准备：沉淀简历表达、项目亮点、技术难点、取舍理由、追问答案。

## 2. 当前项目定位

MyChat 可以定位为：

```text
一个基于 C++ 的分布式 IM 后端项目，包含 Gateway HTTP/WebSocket 统一接入层、
User/Message/Friend/Group/Push 服务边界、gRPC ForwardPacket 远程调用、
Redis 会话与 token 状态、PostgreSQL/ODB 持久化、实时消息推送，
并配套 React Web 客户端用于完整业务流验证。
```

当前最重要的架构事实：

```text
HTTP / WebSocket
-> MessageParser
-> UnifiedMessage
-> MessageProcessor
-> GatewayCommandHandlerRegistry
-> GatewayRuntimeRegistry
-> local service packet dispatcher 或 remote ForwardPacket gRPC
```

后续所有文档和面试表达都必须基于这条链路，不再把旧 `HTTP controller + Local/RemoteClient` 当作当前架构。

## 3. 当前 MVP 能力

已完成：

- 注册、登录、JWT access token、Redis refresh token。
- 用户资料查询、更新、搜索。
- 好友搜索、申请、处理、好友列表、待处理申请。
- 单聊 HTTP/WS 发送、ack、历史、离线拉取、在线 push。
- 群创建、搜索、加入、退出、资料、成员、群消息、群历史、群消息 push。
- Push service、本地 push、远程 push、Gateway callback、fanout policy。
- User/Message/Friend/Group/Push 独立 gRPC 服务边界。
- Gateway local all-in-one 与 remote-all 多进程运行模式。
- 中文 React Web 验证客户端。
- Gateway 重构后完整压测基准。

不作为当前 MVP 阻塞项：

- 富媒体消息、文件上传、对象存储。
- 消息撤回、编辑、删除、全文搜索。
- 好友备注/分组/黑名单完整体系。
- 群主转让、管理员、禁言、入群审核、踢人。
- 多 Gateway 在线路由、服务发现、负载均衡。
- 生产级监控、链路追踪、自动扩缩容。

## 5. 统一分析模板

后续讲任何业务链路，都按这个模板：

```text
用户操作
-> Web 客户端
-> Gateway HTTP/WS
-> MessageParser / UnifiedMessage
-> MessageProcessor
-> GatewayCommandHandlerRegistry
-> command handler 鉴权和补 header
-> local packet dispatcher 或 remote ForwardPacket
-> service 业务逻辑
-> repository/runtime
-> PostgreSQL/Redis/WebSocket 副作用
-> 返回响应或推送
```

## 6. 面试主线

建议主线：

```text
我做的是 C++ 分布式 IM 后端。项目最大的设计点是 Gateway 统一接入 HTTP 和 WebSocket，
内部通过统一消息模型和 cmd_id 注册中心转发 packet。Gateway 不解析业务 payload，
业务解析和规则都下沉到各 service packet dispatcher。这样既能本地 all-in-one 快速开发，
也能通过 gRPC ForwardPacket 切到多进程分布式服务。
```

重点亮点：

- Gateway 统一 packet 架构。
- 鉴权边界和业务边界分离。
- local/remote 运行模式共享 packet 契约。
- Push remote callback 解决 WebSocket session 归属问题。
- 先持久化再 push 的消息可靠性取舍。
- 压测定位和修复过程，有数据支撑。
